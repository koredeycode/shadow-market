# Shadow Market: Local Deployment & Test Protocol

This guide outlines the complete workflow to compile, build, migrate, and test the **Shadow Market** ecosystem across all environments (**Local (5434)**, **Preview (5435)**, and **Preprod (5436)**).

---

## 1. Prerequisites & Infrastructure Initialization

Shadow Market uses an isolated triple-network database strategy. Before testing, clean and start your development infrastructure.

### Clean Restart (Optional but Recommended)
```bash
# Terminate existing containers and clear volumes
docker-compose down -v

# Start infrastructure (Local, Preview, Preprod DBs)
docker-compose up -d
```

---

## 2. Compilation & Build Phase

Ensure all packages (Contracts, SDK, Backend, CLI/TUI, Web) are synchronized with the latest domain refactors (**Bets**, **sm/username**, and **Network Prefixes**).

### Root Build
From the project root:
```bash
npm run build
```

---

## 3. Database Migration & Initialization

Apply the latest schema (including the `bets` table and updated user fields) to all network-specific databases.

### Local (5434)
```bash
export DATABASE_URL="postgresql://postgres:postgres@localhost:5434/shadowmarket_undeployed"
npm run db:push
```

### Preview (5435)
```bash
export DATABASE_URL="postgresql://postgres:postgres@localhost:5435/shadowmarket_preview"
npm run db:push
```

### Preprod (5436)
```bash
export DATABASE_URL="postgresql://postgres:postgres@localhost:5436/shadowmarket_preprod"
npm run db:push
```

---

## 4. Environment-Specific Deployment

Deploy the Shadow Market contracts to your target Midnight network.

### Target: Preprod
```bash
# Ensure your Preprod wallet has tNIGHT tokens from the faucet
cd packages/contracts
NETWORK_ID=preprod npx ts-node src/deployment/deploy.ts
```

### Target: Local (Docker)
```bash
cd packages/contracts
NETWORK_ID=undeployed npx ts-node src/deployment/deploy.ts
```

---

## 5. End-to-End Testing (The Three Heads)

### A. Backend Services (Wager/Market Hub)
```bash
cd packages/backend
# Switch network: undeployed | preview | preprod
MIDNIGHT_NETWORK_ID=preprod npm run dev
```

### B. Web Dashboard (High-Fidelity UI)
Ensure the `VITE_NETWORK_ID` matches your desired environment.
```bash
cd packages/web
VITE_NETWORK_ID=preprod npm run dev
```
**Test Case:**
1. Connect wallet (Check for `mn_addr-preprod...`).
2. Verify autogenerated **`sm/username`** in the TopBar.
3. Open a market, place a **Bet** (Confirm terminology: "Positions" -> "Bets").
4. Navigate to **Portfolio > Bet Details** (Verify P&L and social identity).

### C. Cliff TUI (Power-User Terminal)
```bash
cd packages/cli
# Start TUI pointing to the active backend
MIDNIGHT_NETWORK_ID=preprod npm run dev:tui
```
**Test Case:**
1. Navigate to **Wallet Details**.
2. Verify **Account: `sm/username`** field.
3. Verify **Bech32m Address Prefix** matches the network.

### D. CLI Interaction (Protocol Integration)
```bash
cd packages/cli
# Test Market Creation
MIDNIGHT_NETWORK_ID=preprod ./bin/shadow-market market create --question "Did the build succeed?"
```

---

## 6. Troubleshooting Identity Systems

If **`sm/username`** or **Address Prefixes** are incorrect:
1. **Clear Storage**: `localStorage.clear()` in Browser Console (Web) or delete the TUI session token in `~/.shadow-market`.
2. **Indexer Check**: Verify the `INDEXER_URL` in `packages/backend/src/config.ts` matches the official v4 endpoint.
3. **Codec Check**: Ensure the `ShieldedAddress.codec` in `useWallet.ts` is receiving the correct `NETWORK_ID` string.
